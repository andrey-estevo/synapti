# PRD - Site AAPR Sistemas / Robo Andre

## 1. Visao geral

O projeto consiste em uma landing page para apresentar e comercializar um sistema de atendimento automatico pelo WhatsApp com uso de inteligencia artificial. O site tambem sera usado durante a apresentacao do curso para demonstrar a proposta do sistema e permitir interacao por QR Code.

## 2. Objetivo do produto

Criar uma pagina clara, responsiva e comercial para explicar como o sistema funciona, destacar seus beneficios e direcionar interessados para contato pelo WhatsApp.

## 3. Publico-alvo

- Pequenas e medias empresas que recebem contatos pelo WhatsApp.
- Comercios, academias, prestadores de servico e equipes de vendas.
- Clientes que precisam responder mensagens com mais rapidez.
- Professores e colegas durante a apresentacao do Projeto Integrador.

## 4. Problema que o projeto resolve

Muitas empresas perdem clientes por demora no primeiro atendimento, falta de organizacao nas conversas ou indisponibilidade fora do horario comercial. O sistema proposto automatiza respostas iniciais, melhora a experiencia do cliente e ajuda a transformar interessados em oportunidades de venda.

## 5. Proposta de valor

Automatizar o atendimento pelo WhatsApp com respostas inteligentes, disponibilidade 24 horas e mais chances de converter interessados em clientes.

## 6. Funcionalidades principais do site

- Apresentar a marca AAPR Sistemas.
- Explicar o beneficio principal do sistema logo no topo.
- Mostrar recursos como atendimento 24h, IA inteligente, integracao com WhatsApp e apoio a vendas.
- Explicar o funcionamento em 3 passos:
  - Cliente chama no WhatsApp.
  - Sistema entende a solicitacao.
  - Atendimento acontece automaticamente.
- Apresentar planos comerciais.
- Exibir QR Code para teste e interacao pelo celular.
- Disponibilizar formulario e informacoes de contato.
- Permitir acesso rapido ao WhatsApp por botoes e icone flutuante.

## 7. Requisitos funcionais

- O usuario deve conseguir navegar pelas secoes do site pelo menu.
- O usuario deve conseguir clicar nos botoes de WhatsApp e ser direcionado para conversa.
- O usuario deve conseguir escanear o QR Code para interagir pelo celular.
- O formulario de contato deve abrir o aplicativo de e-mail padrao com a mensagem preenchida.
- O menu deve funcionar corretamente em celulares.
- Os numeros da secao de resultados devem ter animacao ao aparecerem na tela.

## 8. Requisitos nao funcionais

- O site deve ser responsivo para desktop e celular.
- O carregamento deve ser simples e rapido.
- O layout deve transmitir profissionalismo e clareza comercial.
- O site deve funcionar como pagina estatica, sem necessidade de backend.
- O codigo deve usar HTML, CSS e JavaScript puro.

## 9. Tecnologias utilizadas

- HTML5 para estrutura da pagina.
- CSS3 para layout, cores, responsividade e efeitos visuais.
- JavaScript para menu mobile, formulario e animacao dos numeros.
- Font Awesome para icones.
- Git e GitHub para versionamento.
- GitHub Pages para publicacao online.

## 10. Criterios de sucesso

- O site deve abrir corretamente pelo link publico.
- O QR Code deve aparecer de forma legivel.
- A proposta do sistema deve ser entendida nos primeiros segundos.
- O site deve funcionar bem em celular e computador.
- O professor e a turma devem conseguir testar a interacao durante a apresentacao.
- O projeto deve poder ser reaproveitado posteriormente para divulgacao comercial.
